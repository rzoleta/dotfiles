Commit the current changes in the diff. Use conventional commit style for the commit message.

Keep the commit message terse and concise.

Commit body is optional. Skip it when the subject already explains the change. Use a body only for large changes, non-obvious reasoning, breaking changes, migrations, security fixes, or other context future readers will need.
